A few practical notes: Which extension attribute to use — extensionAttribute1 through extensionAttribute15 are all available. Pick one that isn't already in use in your tenant. You can check via PowerShell: powershell Get-MgUser -UserId "user@domain.com" -Property onPremisesExtensionAttributes | Select -ExpandProperty onPremisesExtensionAttributes How to set the value on a user to exclude them: powershell Update-MgUser -UserId "user@domain.com" -OnPremisesExtensionAttributes @{extensionAttribute1 = "NoLicense"} One caveat — -ne in Azure AD dynamic rules also matches users where the attribute is null (not set).
